# Dept
학과(부) 중심의 프라이빗 대학 AI 매칭 플랫폼
기존 대학 커뮤니티의 지나친 개방성을 넘어, 학과(부)라는 가장 밀접한 단위의 소통을 실현합니다.

기존의 대학 커뮤니티 사이트는 지나치게 개방적이고 광범위한 공간 특성을 가지고 있었습니다. 개별 학과의 특성이나 소규모 단위의 구성을 제대로 반영하지 못한다는 한계가 있었으며, 특히 익명성 뒤에 숨은 채로 활동하다 보니 학교 구성원들이 마음 놓고 편안하게 같이 커뮤니케이션을 하고 모임을 할 사람을 찾기 어려웠습니다. DEPT는 이 지점에 주목하여 학우들의 pain point를 해결하고자 합니다.

DEPT의 목표는 학과 구성원이라는 가장 신뢰할 수 있는 단위 안에서, 모임과 학습이 자연스럽게 만들어지는 환경을 만드는 것입니다. 학교 구성원 인증 시스템을 통해 인증된 학과 구성원만 이용 가능하도록 하여, 익명성에서 오는 불안감 없이 편하게 소통할 수 있는 공간을 만듭니다.  LLM 기반 매칭 기능으로 학술 학회, 스터디 그룹, 친목 모임 등 목적에 맞는 사람을 빠르고 정확하게 연결합니다. 직접 모집글을 쓰고 기다리는 대신, AI가 관심사·목표를 분석해 먼저 추천해주는 경험을 제공합니다.

## Minimum Viable Product

### solution

-  모임 및 프로젝트 매칭: 학과 내 학술 학회, 친목 모임, 스터디 그룹 형성을 llm을 통해서 도와줍니다.

### function

- Frontend : Html, css, javascript, RemixIcon(icon library)
- Backend : Python 3.XX, Django 5.X, Gunicorn
- Database : PostgreSQL, Redis
- Infrastructure : Docker Compose

#### function 1 ai matching

DEPT가 제공하는 서비스 기능은 구성원 인증, LLM 기반 사용자 매칭, 내부 매칭방 채팅 세 가지로 구분할 수 있습니다. 구성원 인증 단계에서는 1차로 특정 학교 도메인의 이메일을 입력받아 해당 학교 구성원임을 인증합니다. 현재는 MVP 테스트 단계라 이 수준까지만 인증을 진행하고 있으며, 이후에는 모바일 학생증 OCR 인증과 어드민 승인 절차까지 추가할 계획입니다. LLM 기반 사용자 매칭에서는 Google Gemini API를 활용합니다. 사용자가 자연어로 자신이 원하는 매칭 형태를 서술하면, Gemini API가 반환한 응답을 JSON 형태로 파싱하여 매칭을 진행합니다. 내부 매칭방 채팅 기능에서는 매칭된 사용자들이 서로의 정보를 파악하고 확인할 수 있도록 Django Channels(ASGI)와 Redis Channel Layer를 기반으로 정보를 실시간으로 동기화하며, 그룹 내부에서 채팅을 주고받을 수 있도록 지원합니다.
