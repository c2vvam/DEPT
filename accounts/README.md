# CreditCampus Authentication & Authorization System

이 문서는 CreditCampus 프로젝트의 `accounts` 앱에 구현된 로그인, 회원가입, 로그아웃 기능의 구조 및 세부 보안 설계 사항에 대해 설명합니다.

---

## 1. 구현 기능 개요

- **로그인 (`/accounts/login/`)**: 사용자 이메일과 패스워드를 입력받아 안전하게 인증하고 세션을 생성합니다. '로그인 상태 유지' 선택 여부에 따라 세션 만료 시간을 제어합니다.
- **회원가입 (`/accounts/signup/`)**: 신규 사용자의 이름, 이메일, 패스워드를 입력받아 중복 검사 및 포맷 유효성 검증을 거친 후 계정을 생성합니다.
- **로그아웃 (`/accounts/logout/`)**: 활성화된 세션을 서버 및 클라이언트 측에서 즉각 무효화하고 파기합니다.

---

## 2. 보안 설계 및 방어 체계 (Security Measures)

CreditCampus는 사용자의 개인정보와 세션 안전을 위해 프로젝트 보안 규격(`security.md`)을 철저히 준수하여 설계되었습니다.

### ① SQL Injection 방어
- **상세**: 사용자가 입력한 이메일과 패스워드를 원시 SQL 쿼리 문자열에 가공하여 사용하지 않습니다.
- **구현**: 파라미터화된 쿼리(Parameterized Query)를 기본 실행하는 Django ORM 및 `authenticate()` 인터페이스만을 사용하여 검증을 처리하므로 인젝션 위협을 원천 차단합니다.

### ② Brute-Force (무차별 대입) 공격 방어 & Rate Limiting
- **상세**: 악의적인 사용자가 무차별적으로 로그인 요청을 반복 전송하는 공격을 제한합니다.
- **구현**:
  - Django Cache API를 사용하여 로그인 실패를 기록합니다.
  - 로그인 실패 시 **클라이언트 IP** 및 로그인 시도 **이메일** 각각에 대해 제한 횟수를 누적시킵니다.
  - **5분 이내에 5회 실패** 시 해당 IP 혹은 이메일의 로그인 접수를 **5분(300초)**간 완전히 차단합니다.

### ③ Session Fixation (세션 고정) 방어
- **상세**: 공격자가 피해자의 브라우저 세션 ID를 미리 설정해두고 가로채는 공격을 방어합니다.
- **구현**: Django의 `login(request, user)` 함수는 로그인 성공 즉시 기존 세션 ID를 파기하고 새로운 무작위 세션 키를 생성(Session ID Rotation)하여 공격을 사전에 예방합니다.

### ④ Open Redirect (오픈 리다이렉트) 방어
- **상세**: 로그인 후 이전 페이지로 복귀하게 돕는 `next` 파라미터가 공격용 외부 주소로 변조되어 위장 피싱 사이트로 유도되는 취약점을 차단합니다.
- **구현**: `django.utils.http`의 `url_has_allowed_host_and_scheme`를 호출하여 리다이렉션 경로가 안전한 로컬 호스트 영역인지 및 사용 중인 HTTPS/HTTP 스킴과 일치하는지를 엄격하게 검증합니다. 비정상적인 경로 탐지 시 기본 메인 페이지로 강제 복귀합니다.

### ⑤ 패스워드 일방향 암호화 Hashing
- **상세**: DB 침탈이 발생하더라도 해커가 원본 비밀번호를 탈취할 수 없도록 강도 높은 해시 알고리즘을 사용합니다.
- **구현**: `User.objects.create_user()` 메서드를 호출하여 비밀번호를 안전하게 솔팅(Salting) 및 단방향 암호화 처리합니다. Django의 검증 규격(PBKDF2 등)을 기반으로 고강도로 해싱됩니다.

### ⑥ Account Enumeration (계정 정보 수집) 방어
- **상세**: 특정 사이트에 가입된 회원들의 이메일 리스트를 수집하는 공격을 차단합니다.
- **구현**: 로그인 실패 시 이메일 유무나 비밀번호 오류 여부를 구분하지 않고, 항상 동일한 모호한 피드백(`"이메일 또는 비밀번호가 올바르지 않습니다."`)을 반환합니다.

### ⑦ 세션 쿠키 하이재킹 및 XSS/CSRF 대응
- **상세**: 세션 토큰 및 CSRF 토큰 노출을 물리적으로 차단합니다.
- **구현** (`config/settings.py`):
  - `SESSION_COOKIE_HTTPONLY = True` 및 `CSRF_COOKIE_HTTPONLY = True` 설정을 통하여 자바스크립트(`document.cookie`)의 접근을 차단하고, XSS에 의한 쿠키 탈취를 방지합니다.
  - `SESSION_COOKIE_SAMESITE = 'Lax'`로 Cross-Site 공격 시 전송 제한을 설정합니다.
  - 배포 환경(`DEBUG = False`)인 경우에는 쿠키가 HTTPS 프로토콜 상에서만 전달되도록 `SESSION_COOKIE_SECURE = True`를 자동 강제합니다.

## 3. UI/UX 및 인터랙티브 요소

- **마우스 트래킹 배경 효과**: 로그인/회원가입 카드 뒷단에 마우스의 좌표를 따라 천천히 감속하며 부드럽게 뒤쫓는 반응형 백그라운드 블롭(Blob) 애니메이션이 적용되어 있습니다.
- **비밀번호 보기/숨기기 토글**: 비밀번호 입력 창 내부의 눈 아이콘 버튼을 통해 입력 값을 텍스트 형태로 손쉽게 교차 확인할 수 있습니다.
- **유기적인 에러 피드백 상자**: 잘못 입력하거나 제한 오류 발생 시, Glassmorphic 디자인의 은은한 경고 모듈이 상단에서 슬라이드 다운 애니메이션을 동반하며 부드럽게 등장합니다.
